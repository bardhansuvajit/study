(capture, target, bubble) DOM Event Propagation is the process that determines the order in which event handlers are called when an event (like a click) happens on nested HTML elements. It happens in 3 phases: 1. Capturing Phase/ Capture/ Trickle Down The event starts from the root (window/document) and goes down the DOM tree to the target element. If you attach an event listener with { capture: true }, it runs during this phase. 2. Target Phase The event reaches the target element (the one that was actually clicked or interacted with). Handlers on this element are run. 3. Bubbling Phase/ Bubble Up After the event hits the target, it bubbles back up the DOM tree from the target to the root. By default, most event listeners trigger in this phase. EXAMPLE
OUTPUT outer inner ‚úÖ outer runs first because it's in capture phase, ‚úÖ inner runs in the bubbling phase. You can control event flow using: event.stopPropagation() ‚Üí stops further propagation. event.preventDefault() ‚Üí prevents default browser behavior. How do you optimize DOM manipulation for performance? DOM manipulation is how you make dynamic changes to a web page after it loads. OPTIMIZE Store DOM references in variables ( const button = document.querySelector("#btn"); ) Group multiple changes together instead of one by one. Libraries like React use a Virtual DOM to reduce direct DOM updates. innerHTML is fast but re-renders everything inside ‚Äî avoid using it often. Use textContent, or DOM methods (createElement, appendChild) for small changes. Use CSS animations/transitions instead of JavaScript-based animation when possible ‚Äî they're optimized by the browser.
